📏 STRICT RULES (문제 품질 가드레일 16개)

NEWS-BASED: 실제 뉴스 기반만

POLITICS/WORLD: 5대 고신뢰 소스만
VERIFIABLE EVENTS: 검증 가능한 사실만
OBJECTIVELY MEASURABLE: 수치 기준 필수
SPORTS: 일정 API 데이터만 사용
FUTURE ONLY: 미래 이벤트만
ABSOLUTE UTC TIME: 절대 시각 표기
RESOLVABLE: 마감 전 판정 가능
WITHIN 48H: 48시간 이내
PRICE/MARKET: 24시간 이내 체크
PRICE THRESHOLD: 현재가 ±5% 이내
MINIMUM DEADLINE: 뉴스 문제 24h 이상
ANSWER NOT KNOWN: 답이 알려진 문제 금지
DEATH TOLL: 사망자수 문제 금지 → 후속 이벤트로

v3 — RSS 뉴스 피드 주입 news_feed_service.py
✅ 개선: BBC·CNN·Reuters 등 실시간 헤드라인 → 팩트 기반 문제 강제

v3.1 — 실시간 주가 주입 + URL 매칭 기초 stock_price_service.py

❌ 남은 문제: 단순 문자열 매칭으로 Ford(F), Visa(V) 등 1글자 티커 오매칭
v3.2 — source_url 매칭 완전 개선 gemini_service._resolve_source_url()
✅ 개선: 3단계 우선순위 매칭 — 스포츠→Google검색, 주가/코인→Yahoo Finance, 뉴스→RSS 실제 URL
✅ 개선: 1글자 티커 오매칭 방지 (TICKER_ALIASES + 괄호 안 심볼 정확 매칭)
✅ 개선: POLITICS/WORLD → BBC·Reuters·NYT·AP·Guardian 고신뢰 소스만 사용 (18개 피드)
v3.3 — 판정 기준 명확화 · 종목 확대
✅ 개선: [OBJECTIVELY MEASURABLE] → 수치 기준 필수, 모호한 문제 차단
✅ 개선: 실시간 주가 13개 → 70개 종목 (빅테크·AI·EV·핀테크·코인·지수·원자재·외환)
✅ 개선: 카테고리 이모지 + 우선순위 정렬 (Politics→World→Sports→Tech→Economy)
v3.4 — 마감 시간 · 카테고리 분류 개선
✅ 개선: [MINIMUM DEADLINE] → 뉴스 기반 문제 마감 최소 24시간 이상 강제
✅ 개선: [CATEGORY] → 주가 문제는 무조건 economy (NVDA 주가 → tech 오분류 방지)

✅ 개선: [BANNED PATTERNS] → "official response/statement 발표?", "face X?" 등 판정 불가 패턴 전면 금지
✅ 개선: [DEATH TOLL] → 사망자 수 초과 문제 금지, 후속 이벤트(RSF 재공격? UN 긴급회의?) 유도
