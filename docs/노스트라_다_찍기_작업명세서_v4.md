### 3.5 RPC 함수

**place_bet (베팅 + 서버 시간 검증):**
```sql
CREATE OR REPLACE FUNCTION place_bet(
  p_uid UUID, p_issue_id INTEGER, p_choice VARCHAR(3)
) RETURNS JSONB AS $$
DECLARE
  v_deadline TIMESTAMPTZ; v_status VARCHAR(10);
BEGIN
  SELECT betting_deadline, status INTO v_deadline, v_status
  FROM Issues WHERE issue_id = p_issue_id;

  IF v_status != 'OPEN' THEN
    RETURN jsonb_build_object('success', false, 'error', 'issue_not_open');
  END IF;
  IF now() > v_deadline THEN
    RETURN jsonb_build_object('success', false, 'error', 'deadline_passed');
  END IF;

  INSERT INTO Bets (uid, issue_id, choice) VALUES (p_uid, p_issue_id, p_choice);

  IF p_choice = 'Yes' THEN
    UPDATE Issues SET total_yes = total_yes + 1 WHERE issue_id = p_issue_id;
  ELSE
    UPDATE Issues SET total_no = total_no + 1 WHERE issue_id = p_issue_id;
  END IF;

  UPDATE Users SET last_played_date = now() WHERE uid = p_uid;
  RETURN jsonb_build_object('success', true);
END;
$$ LANGUAGE plpgsql;
```

**resolve_issue (결과 판정 + 스트릭):**
```sql
CREATE OR REPLACE FUNCTION resolve_issue(
  p_issue_id INTEGER, p_correct_answer VARCHAR(3)
) RETURNS JSONB AS $$
DECLARE bet_record RECORD;
BEGIN
  UPDATE Issues SET correct_answer = p_correct_answer, status = 'RESOLVED'
  WHERE issue_id = p_issue_id AND status = 'CLOSED';

  FOR bet_record IN SELECT uid, choice FROM Bets WHERE issue_id = p_issue_id
  LOOP
    IF bet_record.choice = p_correct_answer THEN
      UPDATE Users SET current_streak = current_streak + 1,
        total_wins = total_wins + 1,
        max_streak = GREATEST(max_streak, current_streak + 1)
      WHERE uid = bet_record.uid;
    ELSE
      UPDATE Users SET current_streak = 0 WHERE uid = bet_record.uid;
    END IF;
  END LOOP;

  RETURN jsonb_build_object('success', true, 'resolved_issue', p_issue_id);
END;
$$ LANGUAGE plpgsql;
```

### 5.2 사용자 인증 (Google OAuth via Supabase)

**Flask에서의 OAuth 플로우:**
```
1. 유저가 "Sign in with Google" 클릭
2. Flask → Supabase Auth의 OAuth URL로 리다이렉트
3. 구글 로그인 완료 → Supabase가 콜백 URL로 리다이렉트 (access_token 포함)
4. Flask 콜백 라우트에서 토큰 수신 → Flask session에 저장
5. 최초 로그인이면 Users 테이블에 레코드 생성
```

**routes/auth.py:**
```python
from flask import Blueprint, redirect, request, session, url_for
from services.supabase_client import supabase

auth_bp = Blueprint('auth', __name__)

@auth_bp.route('/login')
def login():
    """Supabase OAuth URL로 리다이렉트"""
    result = supabase.auth.sign_in_with_oauth({
        'provider': 'google',
        'options': {
            'redirect_to': url_for('auth.callback', _external=True)
        }
    })
    return redirect(result.url)

@auth_bp.route('/callback')
def callback():
    """OAuth 콜백 — 토큰 수신 후 세션 저장"""
    # Supabase가 URL fragment로 토큰 전달
    # 클라이언트 JS에서 토큰을 서버로 POST하는 방식 사용
    return render_template('auth_callback.html')

@auth_bp.route('/set-session', methods=['POST'])
def set_session():
    """JS에서 전달받은 토큰으로 세션 설정"""
    access_token = request.json.get('access_token')
    user = supabase.auth.get_user(access_token)

    session['uid'] = user.user.id
    session['email'] = user.user.email
    session['nickname'] = user.user.user_metadata.get('name', 'Anonymous')
    session['access_token'] = access_token

    # 최초 로그인 시 Users 테이블에 레코드 생성
    ensure_user_exists(session['uid'], session['nickname'])

    # 72시간 리셋 체크
    check_streak_reset(session['uid'])

    return jsonify({'success': True})

@auth_bp.route('/logout')
def logout():
    session.clear()
    return redirect(url_for('main.index'))
```

**72시간 리셋 로직:**
```python
from datetime import datetime, timedelta, timezone

def check_streak_reset(uid):
    """앱 진입 시 호출 — 72시간 미참여 시 streak 리셋"""
    user = supabase.table('Users').select('last_played_date').eq('uid', uid).single().execute()
    last_played = user.data.get('last_played_date')

    if last_played:
        last_dt = datetime.fromisoformat(last_played)
        if datetime.now(timezone.utc) - last_dt > timedelta(hours=72):
            supabase.table('Users').update({'current_streak': 0}).eq('uid', uid).execute()
```

### 5.3 다국어 지원 시스템

#### 5.3.1 언어 감지 및 선택

**Flask에서 브라우저 언어 감지 (Streamlit보다 간단!):**
```python
# Flask는 request.accept_languages를 기본 제공
from flask import request

def get_user_language():
    """우선순위: DB preferred_lang > 브라우저 감지 > 기본값 'en'"""
    # 1순위: DB에 저장된 선호 언어
    if 'uid' in session:
        user = get_user(session['uid'])
        if user and user.get('preferred_lang'):
            return user['preferred_lang']

    # 2순위: 브라우저 Accept-Language 헤더
    supported = ['en', 'ko', 'ja', 'zh', 'es']
    browser_lang = request.accept_languages.best_match(supported)

    # 3순위: 기본값 영어
    return browser_lang or 'en'
```

> **vs Streamlit:** Streamlit은 `streamlit-javascript` 외부 패키지 + JS 실행이 필요했지만,
> Flask는 `request.accept_languages` 한 줄로 해결. 별도 패키지 불필요.

```python
        supabase.table('Users').update({'preferred_lang': lang}).eq('uid', session['uid']).execute()
    return jsonify({'success': True})
```

```javascript
async function placeBet(issueId, choice) {
    const btn = document.querySelector(`#bet-${issueId}-${choice}`);
    btn.disabled = true;

    try {
        const res = await fetch('/api/bet', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify({issue_id: issueId, choice: choice})
        });
        const data = await res.json();

        if (data.success) {
            // 버튼 비활성화 + 비율 바 업데이트
            disableBetButtons(issueId);
            updateProgressBar(issueId, data.total_yes, data.total_no);
        } else {
            showError(data.error);
        }
    } catch (err) {
        showError('err_server');
    }
}
```

### 5.6 문제 출제 워크플로우

```
[Gemini 생성] → DRAFT → [관리자 검토/승인] → OPEN → [마감시간 도래] → CLOSED → [결과 판정] → RESOLVED
```

| 단계 | 트리거 | 동작 |
|:---|:---|:---|
| DRAFT | 관리자 Gemini 버튼 클릭 | AI 문제 생성 → DRAFT로 DB 저장 |
| DRAFT → OPEN | 관리자 "승인" 클릭 | status=OPEN, 유저에게 노출 |

```python
# routes/admin.py
from functools import wraps

def admin_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if 'uid' not in session:
            return redirect(url_for('auth.login'))
        user = get_user(session['uid'])
        if user.get('role') != 'admin':
            abort(403)
        return f(*args, **kwargs)
    return decorated

@admin_bp.route('/')
@admin_required
def dashboard():
    drafts = get_issues_by_status('DRAFT')
    closed = get_issues_by_status('CLOSED')
    return render_template('admin/dashboard.html', drafts=drafts, closed=closed)
```

## 6. 에러 처리 정책

| 에러 상황 | 처리 방법 | 사용자 메시지 (영어 기본) |
|:---|:---|:---|
| Supabase 연결 실패 | try-except | ⚠️ Server connection failed. |
| Gemini API 실패 | 재시도 1회 → 에러 | ⚠️ AI analysis failed. |
| Gemini JSON 파싱 실패 | 로그 + 에러 | ⚠️ Cannot process AI response. |
| 번역 실패 | 영어 원본 표시 | (에러 메시지 없이 원본) |
| 마감 문제 베팅 | RPC 거부 | ⚠️ Betting is closed. |
| 중복 베팅 | UNIQUE 위반 | ⚠️ Already voted. |
| 잘못된 문제 ID | 404 | ⚠️ Issue not found. |
| 네트워크 타임아웃 | 5초 timeout | ⚠️ Request timed out. |
| AdSense 실패 | 플레이스홀더 | (회색 박스 표시) |
| 비로그인 베팅 | 401 | ⚠️ Please sign in. |

**원칙:** 모든 에러는 유저 언어로 표시 (실패 시 영어). Flask의 `errorhandler`로 404, 500 에러 페이지도 다국어 처리.

```python
@app.errorhandler(404)
def not_found(e):
    return render_template('error.html', code=404), 404

@app.errorhandler(500)
def server_error(e):
    return render_template('error.html', code=500), 500
```

# 📌 v3 대비 v4 변경 요약

| # | 변경 사항 | 이유 |
|:--|:---|:---|
| 1 | **Streamlit → Flask + Jinja2** | 레이아웃 자유도, AdSense 네이티브 삽입, 유료 전환 비용 ($250 → $7) |
| 2 | 프로젝트 구조 설계 추가 | Blueprint 기반 모듈 분리 (routes, services, templates, static) |
| 3 | 레이아웃: `st.columns` → CSS Grid | 미디어 쿼리로 반응형 직접 제어 |
| 4 | 인증: Streamlit session → Flask session | OAuth 콜백, 세션 관리 명확화 |
| 5 | 언어 감지: `streamlit-javascript` → `request.accept_languages` | 외부 패키지 불필요, 한 줄로 해결 |
| 6 | 베팅: 전체 리런 → AJAX | 페이지 리로드 없이 카드 단위 업데이트 |
| 7 | 광고: `st.components.v1.html()` 우회 → 네이티브 `<head>` 삽입 | 안정성 + AdSense 정책 준수 |
| 8 | 배포: Streamlit Cloud → Render.com | 무료 → $7 점진적 스케일링 |
| 9 | 관리자: `admin_required` 데코레이터 | Flask 표준 패턴 적용 |
| 10 | 에러: Flask errorhandler 추가 | 404, 500 에러 페이지 다국어 |
| 11 | 통합 테스트 시나리오 H, I 추가 | 비로그인 UX, AJAX 동작 검증 |
| 12 | i18n에 에러 메시지 키 추가 | err_server, err_deadline, err_duplicate |
