# [GA] 개발 명세서: 투표 로직, 포인트 시스템, 통계 UI 개선

## 1. 개요
본 명세서는 1) 마감된 투표 참여 방지, 2) 리더보드 포인트 정책 변경(문제당 1포인트, 기본 0포인트), 3) 내 통계(My Stats) 기능 고도화를 위한 개발 계획을 담고 있습니다. 추가로 발생한 `/api/bets/me`의 `limiter.check()` 500 에러는 즉각 조치되었습니다.

## 2. 작업 상세 내용

### 2.1. 마감된 투표 차단 방지 (Frontend & Backend)
- **Frontend (`static/js/app.js`)**: 
  - `loadIssues()` 함수에서 투표 목록 렌더링 시 현재 시간과 비교하여 이슈의 마감 시간(`close_at`)이 지났는지 검사합니다.
  - 마감된 경우 `예/아니오` 투표 버튼을 비활성화(`disabled = true`) 처리.

### 2.2. 리더보드 포인트 정책 변경 (Backend)
- **목표**: 회원가입 시 기본 포인트를 0으로 설정하고, 향후 문제 정답 시 1포인트씩 부여하는 구조 준비. (현재 1000포인트 기본값 표시 됨)
- **Backend (`routes/auth.py`)**: 
  - 신규 가입 시 `supabase.table('users').insert`에서 `points: 0`을 명시적으로 삽입하도록 수정.
- **기존 유저 데이터베이스 수정**: 
  - 이미 1000포인트가 부여된 사용자의 점수를 0점으로 초기화하기 위해 관리자용 1회성 스크립트 작성 및 실행 예정.

### 2.3. 내 통계(My Stats) 기능 고도화 (Backend & Frontend)
- **Backend (`routes/api.py`)**:
  - 신규 API 엔드포인트 `/api/users/me/stats` 추가.
  - 리턴 값: 내 순위(Rank), 연속 정답(Streak), 누적 정답(Wins), 최근 정답을 맞춘 5문제의 기록 리스트 반환.
- **Frontend (`static/js/app.js` & `templates/index.html`)**:
  - 하드코딩된 `#TODO` 순위와 0 연승/승리 부분을 지우고, 백엔드로부터 받은 데이터 렌더링하도록 수정.
  - "My Stats" 영역 하단에 "내가 최근 맞춘 문제 5개" 텍스트와 UI 목록 추가.

## 3. 테스트 계획
- **에러 모니터링**: 수정된 `/api/bets/me`가 500 에러 없이 정상 작동하는지 프론트엔드 통신 확인.
- **투표 버튼 테스트**: 더미 데이터를 활용해 마감된 이슈의 버튼이 클릭 안 되도록 막히는지 체크.
- **데이터 흐름 테스트**: 신규 사용자 로그인 시 0포인트로 시작하는지, 그리고 내 통계 API가 JSON을 에러 없이 내려주는지 점검.
