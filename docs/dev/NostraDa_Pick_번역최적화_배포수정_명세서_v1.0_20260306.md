# NostraDa Pick - 번역 최적화 및 배포 환경 수정 명세서
**버전:** v1.0
**작성일자:** 2026-03-06

## 1. 개요
본 명세서는 실시간 다국어 번역 API 호출로 인한 **비용/성능 낭비 문제를 해결**하고, **무료 클라우드(Render) 배포 환경 최적화** 및 **XSS 취약점 조치**를 위해 진행된 백엔드/프론트엔드 연동 개발 내용을 정의합니다.

## 2. 주요 개발/수정 사항

### 2.1 서버사이드 번역 (DB 직접 저장 통합)
- **문제점:** 사용자가 플랫폼에 접속하고 언어를 변경할 때마다 구글 번역 API가 호출됨 (O(N) 호출 증가).
- **해결 방안:** 
  - `issues` 및 `dev_issues` 테이블에 번역 컬럼 7개(`title_ko`, `title_ja`, `title_de`, `title_fr`, `title_es`, `title_pt`, `title_zh`) 추가.
  - Gemini API를 통해 새로운 이슈가 생성되는 즉시, 구글 번역 API를 서버에서 한 번 호출하여 모든 언어 버전 생성 후 DB 저장.
  - 프론트엔드에서는 번역 API 호출을 제거하고, DB 필드의 값을 `getTranslatedTitle()` 함수로 직접 접근.

### 2.2 폴링/통신 최적화 및 Render 스핀다운 방지
- **폴링 최적화:** `setInterval/setTimeout`으로 진행되는 이슈 자동 갱신(폴링) 시, 베팅 내역(`bets/me`)과 유저 통계(`users/me/stats`) 조회를 스킵하여 무분별한 쿼리 요청 제거.
- **스핀다운 방지:** 서버 갱신(loadIssues) 대기 주기를 `60초`에서 `49초`로 단축. (Render 무료 티어의 50초 유휴 시 스핀다운 강제 종료 현상 방어).

### 2.3 프론트엔드 보안 조치 (XSS 취약점)
- 가입 또는 변경한 닉네임에 악의적 HTML 태그(`<h1>` 등)가 포함될 경우, UI 상에 그대로 렌더링되는 문제 차단.
- 닉네임 문자열에 전역 이스케이프 함수 `escapeHTML()` 적용.

### 2.4 배포(Deployment) 환경 복원 및 보강
- `requirements.txt`에 누락된 리눅스/배포 구동기 패키지 복원
  - 배포용 웹 서버: `gunicorn==23.0.0`
  - 구글 로그인 연동 종속성: `Authlib==1.3.0`, `cryptography==46.0.5`
- Render 서버상에서 `wsgi:app` 이 정상적으로 데몬을 띄울 수 있도록 구성 정리 완료.
