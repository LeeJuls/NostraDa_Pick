# 노스트라 다 찍기 - 스마트 폴링 및 폴백 강화 (빈 화면 버그 해결) 명세서 (v1.0)
*작성일: 2026-03-06*

## 1. 개요 및 문제 시나리오
- **이슈:** 4시간 단위 정각(UTC 0, 4, 8, 12, 16, 20시)에 Gemini API가 이슈를 생성하는 동안 프론트엔드가 이를 기다리지 못하고 "다음 출제 시간(4시간 후)"으로 성급하게 타이머를 갱신해버림. 이로 인해 한 번 생성에 실패하거나 늦게 생성되면 4시간 동안 빈 화면이 유지되는 치명적 버그 발생.
- **원인:**
  1. 프론트엔드(`app.js`)에서 단순히 빈 리스트가 내려오면 곧바로 다음 시간을 가리킴.
  2. 백엔드(`gemini_service.py`)에서 `production` 환경일 때 API 호출 실패 시 에러 처리(폴백)가 막혀 있어 완전히 비어있는 케이스가 생김.

## 2. 해결 및 변경 계획 (개발 내용)
### [수정 1] 프론트엔드 스마트 폴링 도입 (app.js)
- **변경 사항:** `openCount === 0`일 때 무조건 4시간 후를 가리키는 것이 아니라, **현재 시각이 출제 정각 기준 10분 이내**라면 AI가 생성 중이라고 판단.
- **UI:** "🤖 AI가 최신 뉴스를 짚어내는 중입니다... (1분 간격 갱신)" 메시지 표출.
- **로직:** 1분(60초)마다 `loadIssues()`를 재귀 호출하여 느리게나마 백엔드 생성이 완료되는 즉시 화면을 갱신하도록 설계.

### [수정 2] 백엔드 무조건 생성(폴백) 강화 (gemini_service.py)
- **변경 사항:** `_generate_fallback_issues()` 내부의 `production` 차단 로직(더미 반환 안함)을 해제.
- **효과:** Gemini API가 완전 먹통이 되거나 토큰 한도가 초과하더라도, 안전하게 최소한의 더미 데이터 시장이 열려 4시간 동안 유저가 투표를 못하는 최악의 사태 방지.
