# 노스트라 다 찍기 - 이슈 자동 생성 및 실시간 검색 명세서 (v1.0)
*작성일: 2026-02-28*

## 1. 개요 및 문제 시나리오
**문제 1**: 4시간마다 신규 이슈가 생성되어야 하나, 백엔드에 타이머 스케줄링(Cron) 로직이 전혀 구현되어 있지 않음.
**문제 2**: Gemini가 최신 뉴스(예: 트럼프-이란 전쟁)를 알지 못해 과거 데이터 기반의 문제만 생성함.

## 2. 해결 및 변경 계획 (개발 내용)

### [수정 1] 실시간 검색 기능 (Google Search Grounding) 도입
- **대상 파일**: `services/gemini_service.py`
- **변경 사항**: 
  - `google-generativeai` 구글 검색 툴 활성화 (`tools='google_search_retrieval'`).
  - 프롬프트에 **"최신 Breaking News를 검색하여 기반할 것"** 명시 강화.

### [수정 2] 4시간 자동 생성 스케줄러 적용
- **대상 파일**: `app.py`, `requirements.txt`
- **변경 사항**:
  - `Flask-APScheduler` 또는 `APScheduler` 도입.
  - UTC 0, 4, 8, 12, 16, 20 시각에 맞춰 `reset_and_generate()` 로직이 자동으로 트리거되도록 백그라운드 스케줄러 설정.
  
## 3. 테스트 및 검증(QA) 계획
1. **실시간 트렌드 검증**: 강제 1회 실행하여 최신(2026년 현재) 뉴스가 반영된 질문이 도출되는지 확인.
2. **스케줄러 검증**: 주기를 1분으로 줄여 로컬에서 자동 생성 주기가 정상 작동하는지 확인 후 4시간으로 원복.

## 4. 롤백 전략
- 로컬 검증 중 문제 발생 시, Git 커밋 이전 상태 유지. 라이브 서비스 배포는 로컬 검증 100% 통과 후 진행.
