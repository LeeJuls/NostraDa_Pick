# 🚀 [개발 명세서] 노스트라 다 찍기 (Nostradamus Pick-all)

## 1. 프로젝트 개요
* **서비스명:** 노스트라 다 찍기
* **컨셉:** 글로벌 핫이슈(스포츠, 경제, 정치 등)의 단기 결과(Yes/No)를 예측하고 베팅하는 텍스트 기반 시뮬레이션 게임.
* **핵심 가치:** 화려한 그래픽 없이 텍스트와 수치만으로 도파민을 자극하는 빠른 호흡의 예측 랭킹 시스템.

## 2. 기술 스택 (Tech Stack)
* **Frontend/UI:** Streamlit (텍스트 기반 웹/모바일 반응형 구축)
* **Backend/DB:** Supabase (PostgreSQL 기반, 사용자 인증 및 데이터 저장)
* **AI 연동:** Gemini API (Google Search Grounding 활용, 자동 출제)

## 4. 핵심 기능 명세

### 4.1. 사용자 인증 및 랭킹 시스템
* **구글 로그인:** Supabase Auth를 활용하여 이메일 가입 폼 없이 구글 계정으로 연동.
* **지연 평가(Lazy Evaluation) 리셋 로직:**
  * 로그인 및 메인 화면 진입 시 `last_played_date`를 체크.
  * 72시간(3일) 초과 미참여 시 `current_streak`을 0으로 강제 초기화. (`max_streak`은 보존)
* **랭킹 보드 (Top 100):**
  * 정렬 기준: 1순위 `current_streak`(내림차순), 2순위 `total_wins`(내림차순).
  * 1~3등에게는 이모지(🥇, 🥈, 🥉)를 활용하여 시각적 보상 제공.

### 4.2. AI 문제 출제 및 관리자 통제
* **문제 출제:**
  * Gemini API(Search Grounding 포함)를 통해 24~48시간 내 결과가 나오는 이슈 3~5개를 JSON 형식으로 추출.
* **관리자 API 호출 제어 (과부하 방지):**
  * 관리자가 'Start Gemini AI Analysis' 버튼 클릭 시, 즉시 **10초 쿨다운**을 적용하여 버튼을 비활성화.
  * 비활성화 시간 동안 화면에 **토스트 메시지**와 함께 버튼 옆에 **카운트다운 타이머**를 표시하여 중복 호출 차단.

### 4.3. 투표(베팅) 및 어뷰징 방지 로직
* **중복 투표 방지:** 유저가 'Yes' 또는 'No' 베팅을 완료하면 즉시 해당 버튼들을 비활성화하고, 현재까지의 투표 비율(%)을 화면에 렌더링.
* **마감 시간 (Deadline Lock):**
  * 현재 시간이 `betting_deadline`을 초과한 문제는 버튼을 비활성화하고 "[베팅 마감됨]" 표시.
  * 백엔드(Supabase) Insert 시 서버 시간을 기준으로 2차 검증 수행.

## 5. UI/UX 및 에러 처리 정책
* **예외 처리 (Error Handling):** 잘못된 문제 ID를 입력하거나 데이터 검증에 실패했을 때, 화면에 계속 로딩 아이콘을 띄우는 대신 상태를 **'Validation failed'**로 명확히 업데이트하여 노출.
* **면책 조항 (Disclaimer):** 복잡한 법적 문구 대신, 서비스 하단에 **"투자 및 예측 결과에 대한 책임은 전적으로 사용자 본인에게 있습니다."**라고 심플하게 명시.

## 6. 보안 (Security)
* **API Key 관리:** Gemini API 키 및 Supabase 연동 설정값은 코드에 직접 작성하지 않고 별도의 설정 파일(예: `.env`, `secrets.toml`)로 분리.
* **버전 관리 통제:** 해당 설정 파일은 반드시 `.gitignore`에 추가하여 깃허브 등 외부 저장소에 절대 노출되지 않도록 원천 차단.
