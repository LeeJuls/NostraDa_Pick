# NostraDa_Pick 개발 히스토리

**날짜:** 2026년 2월 26일
**주제:** 4명의 가상 개발 팀원(PM, Front, Back, QA) 에이전트 스킬 통합 및 프로젝트 관리 프레임워크 문서화

## 1. 개요 및 목적
- 프로젝트의 확장성과 코드 안정성 강화를 위해 4가지 역할(PM, Front, Back, QA)로 구분된 에이전트 스킬 페르소나 도입.
- 글로벌 가이드(모든 소통 한국어, 개발 전 리뷰(plan), 문서화(dev/history), 단계별 단위 테스트 루프, 깃허브 푸시 제어) 체계를 명문화하여 이후 작업의 일관성을 확립.

## 2. 주요 변경 사항 및 의사 결정 논의 내용

### 2.1 에이전트 역할별 SKILL.md 구축
- `.agent/skills/pm/SKILL.md` 생성: 계획 수립 및 문서(dev/history) 통제 권한 집중.
- `.agent/skills/frontend/SKILL.md` 생성: UI/UX 점검 및 프론트 시각화 구성(패키지 설치 통제 기능 부여).
- `.agent/skills/backend/SKILL.md` 생성: API 통신 로직 및 시스템 오류 방지, 성능 최적화(예외 처리 강화, 잦은 테스트 루프 보장).
- `.agent/skills/qa/SKILL.md` 생성: 시스템 통합 리뷰 및 코드 퀄리티 게이트키퍼 기능 수행.

### 2.2 개발 명세서(Docs/dev) 및 To-do 관리 도입
- 프로젝트 요구사항을 명문화하는 `docs/dev/` 폴더 기반 관리 가이드 적용.
- 완료된 내용을 기록하는 `docs/history/` 문서 생성.
- `docs/dev/NostraDa_Pick_AgentSkills_20260226.md` 형태의 작성 표준 적용.

### 2.3 의사 결정 코멘트
- 기존의 단순 코드 개발 방식에서, **에이전트가 자체적으로 오류를 방어하고 테스트를 보수적으로 진행**하도록 하기 위해 각 역할 파일 상단에 `단위별 테스트 진행 및 사용자 승인 대기` 원칙을 가장 중요하게 명시함.
- 프로젝트 자동 푸시에 의한 사고를 방지하기 위해, 모든 에이전트가 "사용자가 명시적으로 지시할 때만 push한다"는 룰을 강제 적용함.
