# [GA] 히스토리 기록: 투표 로직 강화, 포인트 시스템 리셋 및 내 통계 구현

## 1. 개요
- **일자**: 2026-02-27
- **버전**: v1.0
- **작성자**: Antigravity (GA)
- **주요 내용**: 마감 투표 차단, 1문제당 1포인트 정책 반영, 내 통계 UI 구현 및 API 에러 수정

## 2. 의사 결정 및 논의 사항
- **포인트 정책 변경**: 기존 1000포인트 기본 지급 방식에서 0점 시작으로 변경하기로 결정함. 리더보드의 공정성을 위해 기존 유저들의 포인트를 `reset_points.py` 스크립트를 통해 모두 0으로 초기화함.
- **마감 투표 차단**: 서버 부하 및 투표 무결성을 위해 프론트엔드에서 `close_at` 시간을 체크하여 버튼을 비활성화하고, 백엔드에서도 중복 체크 로직을 강화함.
- **내 통계(My Stats) 데이터 구조**: 사용자가 투표한 이슈 중 'RESOLVED' 상태이며 정답을 맞춘 기록만 필터링하여 Streak(연승) 및 Wins(승리)를 계산하도록 API 로직을 설계함.

## 3. 변경 상세 (Code Changes)
- **Backend**:
  - `routes/api.py`: `/api/users/me/stats` 엔드포인트 신설. `limiter.check()` 문법 오류 수정.
  - `routes/auth.py`: 신규 가입 시 `points: 0` 설정.
  - `reset_points.py`: [NEW] 기존 유저 포인트 초기화 1회성 스크립트.
- **Frontend**:
  - `static/js/app.js`: `loadIssues()`에 마감 체크 로직 추가. `loadMyStats()` 구현 및 UI 렌더링 연동.
  - `templates/index.html`: 내 통계 영역에 Rank, Streak, Wins 및 최근 맞춘 문제 리스트 컨테이너 추가.
