# 🚀 프로젝트 히스토리: 투표 및 통계 고도화, UI 개선 및 글로벌 타이머 추가
**버전**: v1.1.0  
**날짜**: 2026-02-27  

## 📌 주요 의사 결정 및 작업 내역

1. **중복 투표 근본적 해결 (방어 로직)**
   - **배경**: 일부 유저가 동일 이슈에 여러 번 투표하여 포인트가 중복 산정되는 문제 발생.
   - **조치**: 
     - `/bet` API 엔드포인트 내에서 `user_id`와 `issue_id`를 조회하여 이미 투표한 내역이 있는지 조회하는 이중 방어 로직 추가.
     - `schema_init.sql`에 DB 무결성을 위한 `UNIQUE (user_id, issue_id)` 설정 가이드라인 추가.

2. **사용자 경험(UX) UI 개선**
   - **배경**: 로그인 버튼 인지가 어렵고, 선택한 투표 내역의 시각적 표시가 부족함.
   - **조치**: 
     - 상단 네비게이션에 있던 Login/Logout 버튼을 우측 `내 통계` 패널로 이동하여 직관성 확보.
     - 투표한 버튼은 원래의 체크마크 이미지 대신 윈도우 기본 이모지 `✅`를 사용하여 가독성을 높임.
     - 로그아웃 상태에서 투표 시도 시 `로그인이 필요한 서비스입니다.` 브라우저 확인(Confirm) 알림창 팝업 후 구글 리다이렉트 기능 추가 (app.js).

3. **Gemini 예측 문제 스케줄 조정 (6시간 -> 4시간)**
   - **배경**: 문제 생성 주기가 길어 다이나믹한 경험이 떨어짐.
   - **조치**: `gemini_service.py` 내의 `close_at` 타임스탬프 계산식을 `+6 hours`에서 `+4 hours`로 변경. 

4. **실시간 글로벌 출제 대기 타이머 및 스케줄 안내**
   - **배경**: 다음 문제가 언제 나오는지 인지하기 어려움.
   - **조치**: 
     - 메인 페이지에 `문제는 UTC 기준 0, 4, 8, 12, 16, 20시에 출제됩니다.` 안내 박스 추가.
     - 다음 출제 주기까지 남은 시간을 초 단위로 실시간 카운트다운 하는 글로벌 타이머 로직(`app.js`) 구현.
     - 다국어(한국어, 영어, 일본어) 텍스트 사전에 모두 반영(`translations`).

5. **자동 영구 로그인 및 AI 질문 중복 방지**
   - **배경**: 브라우저 종료 시마다 로그아웃됨 & AI가 똑같은 질문을 반복 생성함.
   - **조치**: 
     - Flask 세션 기한을 30일(Permanent Session)로 확장.
     - DB에 존재하는 이전 질문 제목을 추출해 Gemini 프롬프트에 배제(Exclude) 데이터로 주입함으로써 동일 주제 출제 완전 차단.
