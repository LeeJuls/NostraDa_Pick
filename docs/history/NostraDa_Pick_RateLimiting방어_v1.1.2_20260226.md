# NostraDa_Pick 히스토리 - Flask-Limiter를 통한 서비스 가용성 보호 (F5 연타/매크로 방어)
- **작성일**: 2026년 2월 26일
- **버전**: v1.1.2

## 1. 개발 배경 (Background)
앞선 중복 투표 방지 및 Pre-fetch 최적화(v1.1.1)로 버튼 광클릭은 막았으나, **사용자가 웹브라우저의 새로고침(F5)을 고의적으로 연타하거나 새로고침 매크로를 돌릴 경우** 여전히 서버와 DB에 부하가 갈 수 있음. 이를 시스템 레벨에서 감지하고 차단할 수 있는 Rate Limiting 장치가 필요함.

## 2. 의사 결정 논의 사항
- **라이브러리 선정**: Flask 생태계에서 가장 검증된 **`Flask-Limiter`**를 도입하기로 결정.
- **차단 기준 정책**:
    - 일반적 유저의 새로고침은 허용하되, 비정상적인 속도(F5 연타 등)는 즉시 차단.
    - `/api/bets/me` (조회용): 1분에 30회 혹은 초당 2회 초과 시 차단.
    - `/api/bet` (투표 전송용): 1분에 10회 초과 시 차단.
    - 벤(Ban) 상태가 되면 `429 Too Many Requests`를 반환하여 DB 접근 전 단계에서 프로세스를 종료함.

## 3. 핵심 변경 내역 (Changelog)
- **`requirements.txt`**: `Flask-Limiter==3.7.0` 추가 및 설치 완료.
- **`app.py`**:
    - `Limiter` 인스턴스 초기화 및 메모리 기반(storage_uri="memory://") 저장소 설정.
    - 블루프린트(`api_bp`)에서도 접근할 수 있도록 `app.config['LIMITER']`에 등록.
- **`routes/api.py`**:
    - `current_app`을 통해 `Limiter` 객체를 동적으로 호출.
    - `place_bet` 및 `get_my_bets` 함수 진입부에 `limiter.check()` 로직을 심어, 호출 빈도가 임계치를 넘을 경우 즉시 에러 응답을 보내도록 보강함.

## 4. 최종 결과 및 확인
- 이제 페이지 무한 새로고침을 시도하면 특정 횟수 이후 서버에서 요청을 거부하며, 이를 통해 서비스 전체의 가용성과 DB 성능을 안전하게 보호할 수 있게 됨.
- 로그인이 안 된 사용자나 악의적인 봇의 API 스팸을 원천 차단함.
