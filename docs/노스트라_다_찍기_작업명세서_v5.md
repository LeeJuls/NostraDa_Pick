### 3.6 RPC 함수

**place_bet:**
```sql
CREATE OR REPLACE FUNCTION place_bet(
  p_uid UUID, p_issue_id INTEGER, p_choice VARCHAR(3)
) RETURNS JSONB AS $$
DECLARE
  v_deadline TIMESTAMPTZ; v_status VARCHAR(10);
BEGIN
  SELECT betting_deadline, status INTO v_deadline, v_status
  FROM Issues WHERE issue_id = p_issue_id;

  IF v_status != 'OPEN' THEN
    RETURN jsonb_build_object('success', false, 'error', 'issue_not_open');
  END IF;
  IF now() > v_deadline THEN
    RETURN jsonb_build_object('success', false, 'error', 'deadline_passed');
  END IF;

  INSERT INTO Bets (uid, issue_id, choice) VALUES (p_uid, p_issue_id, p_choice);

  IF p_choice = 'Yes' THEN
    UPDATE Issues SET total_yes = total_yes + 1 WHERE issue_id = p_issue_id;
  ELSE
    UPDATE Issues SET total_no = total_no + 1 WHERE issue_id = p_issue_id;
  END IF;

  UPDATE Users SET last_played_date = now() WHERE uid = p_uid;
  RETURN jsonb_build_object('success', true);
END;
$$ LANGUAGE plpgsql;
```

**resolve_issue:**
```sql
CREATE OR REPLACE FUNCTION resolve_issue(
  p_issue_id INTEGER, p_correct_answer VARCHAR(3)
) RETURNS JSONB AS $$
DECLARE bet_record RECORD;
BEGIN
  UPDATE Issues SET correct_answer = p_correct_answer, status = 'RESOLVED'
  WHERE issue_id = p_issue_id AND status = 'CLOSED';

  FOR bet_record IN SELECT uid, choice FROM Bets WHERE issue_id = p_issue_id
  LOOP
    IF bet_record.choice = p_correct_answer THEN
      UPDATE Users SET current_streak = current_streak + 1,
        total_wins = total_wins + 1,
        max_streak = GREATEST(max_streak, current_streak + 1)
      WHERE uid = bet_record.uid;
    ELSE
      UPDATE Users SET current_streak = 0 WHERE uid = bet_record.uid;
    END IF;
  END LOOP;

  RETURN jsonb_build_object('success', true, 'resolved_issue', p_issue_id);
END;
$$ LANGUAGE plpgsql;
```

## 6. 운영 자동화 설계 (🆕 핵심 변경)

### 6.1 운영 모드: 수동 vs 풀오토

관리자가 **딱 한 번** 토글로 모드를 전환:

| 항목 | 수동 모드 (초기) | 풀오토 모드 (안정화 후) |
|:---|:---|:---|
| 문제 출제 | 관리자가 Gemini 버튼 클릭 | cron으로 자동 출제 |
| 문제 승인 | 관리자가 DRAFT 검토 후 승인 | 출제 즉시 OPEN (DRAFT 스킵) |
| 마감 처리 | 관리자가 수동 CLOSED | deadline 도래 시 자동 CLOSED |
| 결과 판정 | 관리자가 correct_answer 입력 | Gemini가 검색 후 자동 판정 |
| 잘못된 문제 | 관리자가 삭제 | 관리자가 즉시 삭제 (이것만 수동) |

### 6.3 풀오토 엔진 (services/auto_engine.py)

**핵심:** Flask 앱과 독립적으로 돌아가는 스케줄러. Render.com의 **Cron Job**으로 실행.

```python
# services/auto_engine.py

def run_auto_cycle():
    """풀오토 1사이클: 매 시간 실행"""
    config = get_system_config()
    if config['auto_mode'] != 'on':
        return  # 수동 모드면 아무것도 안 함

    # Step 1: OPEN → CLOSED (마감 시간 지난 문제)
    auto_close_expired_issues()

    # Step 2: CLOSED → RESOLVED (결과 판정)
    auto_resolve_closed_issues()

    # Step 3: 새 문제 출제 (OPEN 문제가 부족하면)
    auto_generate_issues(config)


def auto_close_expired_issues():
    """betting_deadline이 지난 OPEN 문제를 CLOSED로 전환"""
    supabase.table('Issues') \
        .update({'status': 'CLOSED'}) \
        .eq('status', 'OPEN') \
        .lt('betting_deadline', datetime.now(timezone.utc).isoformat()) \
        .execute()


def auto_resolve_closed_issues():
    """
    CLOSED 문제에 대해 Gemini로 결과 확인 후 자동 판정.
    Gemini가 확신도 90% 이상일 때만 자동 판정.
    확신도 낮으면 스킵 (관리자가 수동 처리).
    """
    closed = supabase.table('Issues') \
        .select('*').eq('status', 'CLOSED').execute()

    for issue in closed.data:
        result = gemini_check_result(issue)

        if result['confidence'] >= 0.9:
            # 높은 확신 → 자동 판정
            supabase.rpc('resolve_issue', {
                'p_issue_id': issue['issue_id'],
                'p_correct_answer': result['answer']
            }).execute()
        else:
            # 낮은 확신 → 스킵 (로그만 남김)
            log(f"Issue #{issue['issue_id']}: confidence {result['confidence']}, skipped")


def auto_generate_issues(config):
    """OPEN 문제가 부족하면 자동 출제"""
    open_count = supabase.table('Issues') \
        .select('issue_id', count='exact').eq('status', 'OPEN').execute()

    target = int(config['auto_issue_count'])
    needed = target - open_count.count

    if needed > 0:
        issues = gemini_generate_issues(needed)
        for issue in issues:
            issue['status'] = 'OPEN'  # 풀오토는 DRAFT 스킵, 바로 OPEN
            issue['source'] = 'auto'
            issue['betting_deadline'] = (
                datetime.now(timezone.utc) +
                timedelta(hours=int(config['auto_deadline_hours']))
            ).isoformat()
            supabase.table('Issues').insert(issue).execute()


def gemini_check_result(issue):
    """
    Gemini에게 문제의 결과를 물어봄.
    Search Grounding으로 최신 정보 기반 판정.
    """
    prompt = f"""
    The following prediction question's deadline has passed.
    Check if the result is now known.

    Question: {issue['question_title']}
    Resolution criteria: {issue['resolution_criteria']}

    Respond ONLY in JSON:
    {{
        "answer": "Yes" or "No" or "Unknown",
        "confidence": 0.0 to 1.0,
        "reasoning": "brief explanation"
    }}

    If the result is not yet clear, set answer to "Unknown" and confidence to 0.
    """
    # Gemini API 호출 후 JSON 파싱
```

**Gemini 결과 판정 안전장치:**
```
confidence >= 0.9 → 자동 RESOLVED
confidence < 0.9 → 스킵, 관리자에게 알림 (or 다음 사이클에 재시도)
answer = "Unknown" → 스킵 (결과가 아직 안 나온 경우)
```

### 6.4 문제 즉시 삭제 (🆕 긴급 대응)

잘못된 문제는 모드와 관계없이 **항상** 관리자가 즉시 삭제 가능:

```python
# routes/admin.py
@admin_bp.route('/delete-issue/<int:issue_id>', methods=['POST'])
@admin_required
def delete_issue(issue_id):
    """OPEN 문제 즉시 삭제 — 관련 베팅도 함께 삭제"""
    # 1. 해당 문제에 베팅한 유저들의 베팅 기록 삭제
    bets = supabase.table('Bets').select('uid, choice') \
        .eq('issue_id', issue_id).execute()

    # 2. Bets 삭제
    supabase.table('Bets').delete().eq('issue_id', issue_id).execute()

    # 3. Issue_Translations 삭제
    supabase.table('Issue_Translations').delete().eq('issue_id', issue_id).execute()

    # 4. Issue 삭제
    supabase.table('Issues').delete().eq('issue_id', issue_id).execute()

    return jsonify({'success': True, 'deleted_bets': len(bets.data)})
```

**삭제 정책:**
- 삭제된 문제의 베팅은 "없던 일"로 처리 (streak에 영향 없음)
- 삭제 시 확인 모달 표시 ("이 문제에 XX명이 베팅했습니다. 삭제하시겠습니까?")
- RESOLVED 문제는 삭제 불가 (이미 streak에 반영됨)

### 6.5 Cron Job 설정 (Render.com)

# render.yaml
services:

**가정:**
- AdSense 평균 CPM (1000회 노출당): $1.5 (글로벌 평균, 보수적)
- 문제 2개당 광고 1개 = 유저 1명 방문 시 평균 광고 1.5회 노출 (문제 3개 기준)

**손익분기 계산:**

```
필요 월 수익: $13 (최대 비용 기준)
필요 월 노출: $13 ÷ ($1.5 / 1000) = 약 8,667회
필요 일 노출: 8,667 ÷ 30 = 약 289회
유저 1명당 노출: 1.5회 (광고 배너)
필요 DAU: 289 ÷ 1.5 = 약 193명
```

### 7.3 결론

> **DAU 193명이면 서버비를 커버한다.**

이건 충분히 현실적인 숫자야. 참고로:
- 무료 티어(Render 슬립 모드)로 시작하면 **비용 $0**으로 초기 유저 모집 가능
